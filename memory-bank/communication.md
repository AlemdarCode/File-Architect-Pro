# Communication Preferences

## User Rules
*   **Conciseness**: Do NOT include internal reasoning, "thinking out loud", or detailed analysis of user intent in the final response.
*   **Directness**: State only the necessary information (what was done, essential warnings).
*   **Output**: Focus on the code changes and the result.

*   Avoid reproducing the chain of thought.
